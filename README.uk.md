## Що вміє MedAlarm

- ⏰ Надсилає нагадування за розкладом, заданим користувачем.
- ✅ Дає змогу позначити приймання, пропустити його або відкласти нагадування безпосередньо в Telegram.
- 💊 Зберігає список ліків, кількість для одного приймання, коментарі та кілька часів на день.
- 🔎 Підтримує ручне введення та пошук в офіційному Державному реєстрі лікарських засобів України.
- 📊 Показує план на сьогодні та статистику дотримання розкладу.
- 📜 Групує історію за днями й ліками, підтримує фільтри періоду та статусу.
- 🌐 Працює російською, українською та англійською мовами.
- 🔤 Підтримує три розміри тексту, тактильний відгук і голосове введення там, де це доступно.
- 🔄 Зберігає ліки локально та синхронізує їх із сервером після авторизації через Telegram.
- 📨 Приймає оцінки та повідомлення про помилки, зокрема з необов'язковим знімком екрана.

## Як почати користуватися

1. Відкрийте [@med_alarm_bot](https://t.me/med_alarm_bot?start=app) і натисніть **Start** або надішліть `/start`.
2. Натисніть **«Відкрити MedAlarm»** у повідомленні бота.
3. На вкладці **«Ліки»** натисніть `+`.
4. Виберіть спосіб додавання:
   - **з каталогу МОЗ** — знайдіть зареєстрований препарат, а потім самостійно вкажіть призначену кількість і час;
   - **вручну** — самостійно заповніть назву, кількість, коментар і розклад.
5. Збережіть ліки. Вони з'являться в плані на сьогодні та синхронізуються із сервером.
6. Коли надійде справжнє нагадування, виберіть **«Прийнято»**, **«Пропустити»** або **«Нагадати пізніше»**.
7. Використовуйте вкладку **«Історія»**, щоб переглядати виконані та пропущені приймання.

Чернетка форми зберігається автоматично. Можна залишити сторінку, перемкнутися між ручним введенням і каталогом та повернутися без втрати введених даних.

## Як це працює

MedAlarm складається з чотирьох частин:

- **Telegram-бот** на `aiogram 3` реєструє користувача та доставляє нагадування.
- **Планувальник** на APScheduler створює завдання за збереженим розкладом і відновлює відкладені нагадування після перезапуску.
- **Backend API** на FastAPI та async SQLAlchemy перевіряє Telegram `initData`, синхронізує ліки й обслуговує налаштування, план, історію та зворотний зв'язок.
- **Telegram Mini App** на React, TypeScript і Vite надає мобільний інтерфейс.

Основні дані зберігаються в SQLite. Ліки синхронізуються за моделлю local-first із розв'язанням конфліктів за часом останньої зміни. Справжня історія приймань залишається серверною та завжди прив'язана до події нагадування.

## Локальний запуск

### Варіант 1: інтерфейс із тестовою авторизацією

Потрібні Node.js 22+ і npm:

```powershell
cd frontend
npm install
npm run dev:local
```

Відкрийте `http://localhost:5173/`. Локальний preview використовує тестову Telegram-ідентифікацію та дає змогу ввімкнути ізольований демо-режим. Демо-дані ніколи не надсилаються як справжня історія приймань.

### Варіант 2: бот, API та планувальник у Docker

1. Скопіюйте `.env.example` у `.env`.
2. Укажіть щонайменше дійсний `BOT_TOKEN`, безпечний `JWT_SECRET`, HTTPS-адресу Mini App і дозволені CORS origins.
3. Запустіть стек:

```powershell
docker compose up --build -d
```

Docker запускає API та спільний процес бота з планувальником. Для production-профілю також доступний Cloudflare Tunnel.

### Варіант 3: окремі процеси для розробки

Потрібен Python 3.12+:

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python -m app.catalog_update
uvicorn app.api.main:app --reload --host 0.0.0.0 --port 8000
```

В окремих терміналах:

```powershell
python -m app.bot_main
python -m app.scheduler
```

Для звичайного локального запуску бота разом із планувальником можна використати `python main.py`.

## Перевірка змін

```powershell
python -m pytest
cd frontend
npm run test:local
npm run build
```

CI додатково збирає Docker-образ, перевіряє production Compose і публікує Mini App на GitHub Pages після змін у `main`.

## Каталог ліків

Пошук використовує відкриті дані Державного реєстру лікарських засобів України з порталу [data.gov.ua](https://data.gov.ua/) на умовах **CC BY**. Каталог надає лише довідкові відомості: назву, форму випуску, склад, виробника, реєстраційні дані, ATC і посилання на офіційну інструкцію. Кількість і розклад завжди вводить користувач.

Оновити локальну копію каталогу:

```powershell
python -m app.catalog_update
```

## Розгортання та обслуговування

Production використовує один backend-контейнер на VPS, постійний SQLite volume, Telegram long polling і Cloudflare Tunnel. Не запускайте кілька backend-реплік, поки застосунок використовує SQLite та єдиний процес Telegram polling.

Інструкції з першого розгортання, релізів, резервного копіювання, відновлення та відкочування містяться в [`deploy/README.md`](deploy/README.md).

## Структура проєкту

```text
app/
├── api/          # FastAPI API та Telegram-автентифікація
├── database/     # SQLAlchemy-моделі, сесії та SQLite-міграції
├── handlers/     # /start, /app і дії з нагадувань
├── scheduler/    # завдання APScheduler і доставка нагадувань
└── services/     # бізнес-логіка та запити до бази даних
frontend/         # React/Vite Telegram Mini App
tests/            # backend-тести
frontend/tests/   # тести чистої frontend-логіки
deploy/           # релізи, backup, restore і production-інструкції
```

## Ліцензії та відповідальність

Вихідний код проєкту поширюється відповідно до файлу ліцензії репозиторію, якщо він присутній. Дані українського Державного реєстру використовуються з обов'язковою атрибуцією за ліцензією CC BY.

MedAlarm — інструмент для організації нагадувань, а не медична інформаційна система. З питань призначення, зміни або припинення лікування необхідно звертатися до кваліфікованого медичного фахівця.
